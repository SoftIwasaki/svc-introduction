Structured Vibe Coding

AIのスピードを活かしながら、商用品質の統制を維持する開発フレームワーク。
CLAUDE.mdという「プロジェクト憲法」がAIの振る舞いを宣言的に制御する。

CLAUDE.md = 憲法 3 Gates Cascade Generation Context Isolation

Conceived by Tatsuya Iwasaki — SVC v3.0

01 — THE PROBLEM

AIに「お願い」しても品質は保証されない

設計崩壊・暗黙の仕様補完

AIが勝手にファイルを追加し設計意図が消失する。指示が曖昧だとAIが仕様を推測で埋め、なぜその実装になったか誰もわからない。

品質のギャンブル・判断の消失

品質はプロンプトの書き方に依存し「動いたからOK」が常態化。設計判断の根拠が記録されず、同じ問題が繰り返される。

SVCはこの問題を3つの仕組みで解決する。
プロジェクト憲法（CLAUDE.md）で宣言的にAIを制御し、3つのゲートで物理的に暴走をブロックし、カスケード生成でロードマップから実装指示を自動導出する。
02 — THE VISION

要件定義後は、スマホで番号を選ぶだけ

人間がコードを書く必要はない。設計書を手作業で書く必要もない。SVCが目指すのは、要件定義後の開発プロセス全体を、意思決定番号の入力だけで完結させること。

1

Orchestratorが分析

要件から自動でバッチ分割。設計判断が必要な箇所を選択肢として提示。

2

スマホで番号を選択

Claudeアプリで「Aを選択」と入力するだけ。DECISIONS.mdに自動記録。

3

AIが自律実行

prompt.md → 実装 → テスト → レビュー → report.md を自動で実行。

4

Release Gate GO

全テスト結果を確認し最終GOを出す。ここだけはHuman必須。

これは最終到達像であり、現時点では一部工程で実証済み。
小規模開発（mochica_bip）で商用レベルに到達。現在、中規模開発（BIP Cost Insight: 設計文書71本・22本の実装指示プロンプト）でこのビジョンを検証中。
03 — CASCADE GENERATION

カスケード生成 — SVCの心臓部

ロードマップから実装指示が自動的にカスケードする。AIが「何を作るか」を考えるのではなく、構造化されたファイルから「何を作るべきか」が導出される。

04 — CONTEXT ISOLATION

コンテキスト分離 — 中規模開発を可能にする鍵

AIのコンテキストウィンドウは有限。カスケード構造がコンテキスト消費を最小化し、中規模でも自律実行を持続させる。

05 — HUMAN ROLE

人間は「選ぶ」だけ

設計・実装・テスト・レビューはAIが実行する。人間の役割は「Orchestratorが提示する選択肢から選ぶ」ことに集約される。

## トリアージ結果 判定: Clarification 理由: Azure App Service Planの配置戦略 選択肢: A: BIPと相乗り ($0増) B: 独立Plan (+$13/月) 推奨: A 不確実性: CPU負荷は実測後再判断 ─── 人間が「A」と返すだけ ─── # → DECISION-0007 として自動記録 decision: "BIPと相乗り" rationale: "月額増分$0" revisit: "CPU 70%超が1週間継続なら"

業務判断の選択

「A or B？」に答えるだけ。技術的判断はOrchestratorが選択肢に含めて提示する。

ゲートGO判定

Reviewerのレビュー結果を見て GO / NO GO。証跡はreview.jsonに永続化。

Release Gate最終承認

全テスト結果と品質指標を確認。ここだけは委譲レベルに関わらずHuman必須。

統制の段階性

すべての変更を同じ強度で扱わない。影響の大きさでモードを選択し、委譲マトリクスでAIの自律範囲を制御する。

Lite SVC

10 — IMMUTABLE PRINCIPLES

不変の原則

01Plan First実装より計画を優先する
02Human Judgment Only最終判断は人間のみ。ただし「選ぶだけ」でよい構造を作る
03Gated Orchestration3つのゲートでAIの暴走を機械的に防ぐ
04Cascade Generationロードマップから実装指示を自動導出する
05Context Isolationバッチ単位でコンテキストを分離し、中規模でも持続可能に
06Traceable Decisionsすべての判断を記録する
07CLAUDE.md = Constitutionすべての役割はプロジェクト憲法に従う

SVCは「AIを信頼しない」フレームワークではない。
「AIを信頼できる構造を作る」 フレームワークだ。
